# Running sjmmie-test
# LD_LIBRARY_PATH needs to be set so we can load libjvm.dylib automatically.  If we don't we get this error message:
#
# dyld: Library not loaded: @rpath/libjvm.dylib
#   Referenced from: /Users/timmattison/Desktop/annex/github/sjmmie/./sjmmie-test
#   Reason: image not found
# Trace/BPT trap: 5
#
# SJMMIE_JVM_OPTS needs to be set to tell the JVM where to load its classes from.  If you do not set it and the classes
#   can't be found you'll end up getting this error message:
#
# Segmentation fault: 11

SJMMIE_JVM_OPTS=-Djava.class.path=target/classes SJMMIE_CLASS=com/timmattison/sjmmie/Sjmmie LD_LIBRARY_PATH=/Library/Java/JavaVirtualMachines/jdk1.7.0_45.jdk/Contents/Home/jre/lib/server ./sjmmie-test

# Running the trial code
MAIN=com.timmattison.sjmmie.Sjmmie ./run.sh
